[![Actions Status](https://github.com/Slovuan-Swan/frontend-project-11/actions/workflows/hexlet-check.yml/badge.svg)](https://github.com/Slovuan-Swan/frontend-project-11/actions)

Веб-приложение для агрегации, парсинга и удобного чтения RSS-лент в реальном времени. Проект ориентирован на построение масштабируемой архитектуры без использования высокоуровневых абстракций управления асинхронностью (`async/await`).

## 🛠 Технологический стек и инструменты

- **Язык:** JavaScript (ES6+), архитектура построена строго на нативных **Promise API**
- **Управление состоянием (State):** `Valtio` (Vanilla API) для реактивного связывания данных
- **Сборщик фронтенда:** `Vite`
- **Валидация данных:** `Yup` (Асинхронные схемы валидации)
- **Интернационализация (i18n):** `i18next` (Полная локализация интерфейса и системных ошибок)
- **Сетевые запросы:** `Axios` + интеграция с внешними CORS-прокси сервисами
- **Интерфейс и стили:** `Bootstrap 5` (Модальные окна, адаптивная сетка, компоненты форм)
- **Качество кода:** `ESLint`, автоматический пайплайн `GitHub Actions` (CI)

## 🏗 Архитектурные особенности проекта

Приложение спроектировано по канонам паттерна **MVC (Model-View-Controller)**, разделяя данные, интерфейс и бизнес-логику:

1.  **State (Model):** Единый источник правды, изолированный от DOM-дерева. Хранит нормализованные структуры данных (фиды, посты) и состояние UI (ID открытых постов, прочитанные статьи). Обернут в прокси-объекты библиотеки `Valtio`.
2.  **View (Render layer):** Чистый слой отображения. С помощью `subscribe` слушает изменения состояния и точечно обновляет DOM (переключает CSS-классы, рендерит списки, наполняет модальные окна). Не содержит логики валидации или сетевых вызовов.
3.  **Controller / Parser:** Обработчики событий инициализируют асинхронные цепочки промисов. Входящий поток XML парсится с помощью нативного `DOMParser` внутри изолированной чистой функции, которая выбрасывает типизированные исключения при сбое разметки.

## ⚡ Ключевой функционал

- **Валидация в реальном времени:** Проверка корректности URL, обязательного заполнения полей и исключение дубликатов (повторного добавления одного и того же фида).
- **Автономное обновление (Polling):** Фоновый рекурсивный опрос добавленных источников каждые 5 секунд с дедупликацией по уникальным ссылкам постов (`link`). Свежие публикации плавно добавляются наверх без перезагрузки интерфейса.
- **Интернационализация:** Отсутствие захардкоженного текста на UI. Ошибки валидации Yup, ошибки сети Axios и системные уведомления выводятся через словарь `i18next`.
- **Интерактивный UI:** Просмотр краткого содержания статьи во всплывающем модальном окне Bootstrap с возможностью перехода на первоисточник. Автоматический трекинг и визуальное выделение прочитанных публикаций.

## 🚀 Быстрый запуск (Локально)

Убедитесь, что на вашем компьютере установлен **Node.js** (версии 18 и выше).

1. Клонируйте репозиторий:

   ```bash
   git clone https://github.com
   cd flow-rss
   ```

2. Установите зависимости проекта:

   ```bash
   make install
   ```

3. Запустите локальный сервер для разработки:

   ```bash
   make develop
   ```

   _Приложение будет доступно по адресу `http://localhost:5173/`_

4. Собрать production-версию проекта:

   ```bash
   make build
   ```

5. Проверить код линтером на соответствие стандартам:
   ```bash
   make lint
   ```
